Overview

API Reference Notes

Shared rules used across all routes.

Base URL

All endpoints are relative to the current server origin.

Authentication

Protected endpoints require a JWT in the Authorization header.

Authorization: Bearer <jwt-token>

Response Shape

Success: returns JSON with success: true.
Error: returns JSON with success: false and a message.

Interactive Testing

Select an endpoint to open its tester. /api/health opens by default.

Each response stays with the endpoint that created it.

Directory

Endpoints

Select a route to open its details.

GET /api/health

Checks whether the backend is responding and returns a timestamped health payload.

POST /api/auth/register

Creates a user account and returns a token for immediate authenticated testing.

POST /api/auth/login

Authenticates an existing user and refreshes the session token on this page.

GET /api/auth/me

Fetches the currently authenticated user using the token stored in the session panel.

POST /api/auth/logout

Validates the active session and clears the local token after a successful response.
